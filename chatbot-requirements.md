## 1. 프로젝트 개요

### 1.1 배경

air 고객센터 인입 트래픽 중 자주 묻는 질문(FAQ) 유형이 반복적으로 발생하고 있으며, 이를 챗봇 시나리오로 자동화하여 고객 스스로 답변을 탐색할 수 있는 환경을 제공하고자 한다.

### 1.2 목적

- FAQ 카테고리를 구조화된 시나리오(3단계 Depth)로 제공하여 고객 자가 해결율 향상
- 실시간 상담사 연결 전 사전 안내를 통한 상담 품질 개선
- 채팅 상담 채널 내 FAQ 자동 응대로 상담사 업무 부하 감소

### 1.3 서비스 범위

- **채널**: air 웹사이트 채팅 상담 팝업
- **대상**: air 서비스 가입 고객 및 가입 예정 고객
- **제공 방식**: 시나리오형 FAQ 챗봇 (자유 입력 미지원)

## 3. 범위 정의

### 3.1 In-Scope (구현 대상)

- FAQ 대주제 캐러셀 UI (1 Depth)
- 세부 질문 선택 UI (2 Depth)
- 상세 답변 제공 및 더보기 기능 (3 Depth)
- 채팅 상담사 연결 버튼 및 URL 연동
- 이전으로 / 처음으로 네비게이션
- 카테고리별 FAQ 데이터 (air 회원 가입, 포인트 적립·사용, 요금 납부·해지)

### 3.2 Out-of-Scope (구현 제외)

- 자유 텍스트 입력 기반 챗봇 응답
- 회원 인증 및 개인화 서비스
- 실시간 챗봇 NLP 엔진 연동
- 상담 이력 저장 및 조회
- 관리자 FAQ 편집 UI

## 6. FAQ 콘텐츠 요구사항

|----------|---------|------------|
| air 회원 가입 | 3 | ✅ 확정 |
| 포인트 적립·사용 | 3 | ✅ 확정 |
| 요금 납부·해지 | 3 | ✅ 확정 |
| eSIM·USIM | 3 | ⏳ 미확정 |

### 6.2 콘텐츠 관리 요구사항

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
